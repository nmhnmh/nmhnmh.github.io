title: 回到Github
date: 2019-09-08
tags:
  - Github
---

在过去的几年里，尝试使用了各种高代码托管平台和服务：Coding.net, Gitlab.com, Bitbucket等等，比较之后觉得还是Github最优秀，[Github被微软收购后](https://news.microsoft.com/2018/06/04/microsoft-to-acquire-github-for-7-5-billion/)，本身也发生了非常大的变化，不管是[付费计划的功能升级](https://github.blog/2019-01-07-new-year-new-github/)还是最新的[Github Actions](https://github.com/features/actions)的加入，都是让人欣喜的变化，对于用户来说越来越友好，使用方便，功能日渐强大。

现在Github有了自带的Github Actions，可以用来实现CI/CD，几乎可以实现提交后实时触发执行，响应速度和执行性能都比之前使用第三方(如TravisCI)服务有了明显的提升。功能上也更为强大，比如原生支持macOS/Linux/Windows三个主流操作系统运行环境，这一点就非常了不起。
