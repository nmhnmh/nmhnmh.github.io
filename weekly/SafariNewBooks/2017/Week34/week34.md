<div class="safaribook"><h2><a href="https://www.safaribooksonline.com/library/view/azure-serverless-computing/9781788390828/">Azure Serverless Computing Cookbook</a></h2><p><img src="http://www.safaribooksonline.com/library/cover/9781788390828/" alt="Azure Serverless Computing Cookbook"><ul><li>Authors: <strong>Praveen Kumar Sreeram</strong></li><li>Topic: <strong>Azure</strong></li><li>ISBN: <strong>9781788390828</strong></li><li>Publisher: <strong>Packt Publishing</strong></li></ul>
This book is about <b>Serverless Computing</b>, it is inspired by event-driven programming model, you register some endpoints or cloud functions with your cloud providers, then these endpoints could be called or triggered by external or internal events, it could be a web request, scheduled event, file event or messages etc. It simplifies the development and operating process, you don't need to setup your own servers and runtime environments, you just register your code, and you don't need to manage or maintain your servers, and it is scalable by nature, of course, every coin has two sides, it does sacrifice some flexibility, some features are not available in serverless environments, nowadays serverless computing is supported by major cloud infrastructure providers like Amazon(AWS Lambdas) Google(Cloud Functions), of course Microsoft Azure(Azure Functions). When used with Cloud Storage or Cloud Database, serverless apps could be very powerful.
</p></div>
